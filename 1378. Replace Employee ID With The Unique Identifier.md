# 1378. Replace Employee ID With The Unique Identifier

## Concept
This problem tests your understanding of **LEFT JOIN**, one of the most commonly asked SQL interview topics.

The goal is to retrieve every employee from the `Employees` table and replace their employee `id` with the corresponding `unique_id` from the `EmployeeUNI` table. If an employee does not have a matching `unique_id`, return `NULL`.

---

## Concepts Tested
- `LEFT JOIN`
- Matching rows using a common column
- Handling missing matches (`NULL`)
- Selecting required columns

---

## SQL Keywords Used
- `SELECT`
- `FROM`
- `LEFT JOIN`
- `ON`

---

## Solution

```sql
SELECT eu.unique_id, e.name
FROM Employees e
LEFT JOIN EmployeeUNI eu
ON e.id = eu.id;
```

---

## Explanation

### Step 1: Start with the Employees table

```sql
FROM Employees e
```

- `Employees` is the main table.
- `e` is an alias used to write shorter queries.

Current data:

| id | name |
|----|------|
|1|Alice|
|7|Bob|
|11|Meir|
|90|Winston|
|3|Jonathan|

---

### Step 2: Perform a LEFT JOIN

```sql
LEFT JOIN EmployeeUNI eu
```

- Join the `EmployeeUNI` table.
- `eu` is an alias.

A **LEFT JOIN** keeps **all rows from the left table** (`Employees`).

---

### Step 3: Specify the matching condition

```sql
ON e.id = eu.id
```

Match employees having the same `id`.

Example:

Employees

| id | name |
|----|------|
|1|Alice|
|7|Bob|
|11|Meir|

EmployeeUNI

| id | unique_id |
|----|-----------|
|3|1|
|11|2|
|90|3|

Matching:

- Alice â†’ No match â†’ `NULL`
- Bob â†’ No match â†’ `NULL`
- Meir â†’ Match â†’ `2`

---

### Step 4: Select required columns

```sql
SELECT eu.unique_id, e.name
```

Output:

| unique_id | name |
|-----------|------|
|NULL|Alice|
|NULL|Bob|
|2|Meir|
|3|Winston|
|1|Jonathan|

---

## Why LEFT JOIN?

The question says:

> If a user does not have a unique ID, return `NULL`.

Only a **LEFT JOIN** guarantees that all employees appear in the result.

If we used an **INNER JOIN**, employees without a matching `unique_id` would be excluded.

---

## JOIN Comparison

### INNER JOIN

Returns only matching rows.

Example:

Employees

| id | name |
|----|------|
|1|Alice|
|7|Bob|
|11|Meir|

EmployeeUNI

| id | unique_id |
|----|-----------|
|11|2|

Result:

| unique_id | name |
|-----------|------|
|2|Meir|

Alice and Bob are removed.

---

### LEFT JOIN

Returns **all rows from the left table**.

Result:

| unique_id | name |
|-----------|------|
|NULL|Alice|
|NULL|Bob|
|2|Meir|

## Important Syntax

### LEFT JOIN

```sql
SELECT columns
FROM table1
LEFT JOIN table2
ON table1.column = table2.column;
```

---

### INNER JOIN

```sql
SELECT columns
FROM table1
INNER JOIN table2
ON table1.column = table2.column;
```

---

### Using Table Aliases

```sql
Employees e
EmployeeUNI eu
```

Instead of writing:

```sql
Employees.id
EmployeeUNI.unique_id
```

You can write:

```sql
e.id
eu.unique_id
```

This improves readability, especially in queries with multiple joins.

---

## SQL Execution Order

Although we write:

```sql
SELECT
FROM
LEFT JOIN
ON
```

SQL executes roughly in this order:

1. `FROM`
2. `JOIN`
3. `ON`
4. `SELECT`

---

## Time Complexity

Let:

- **n** = rows in `Employees`
- **m** = rows in `EmployeeUNI`

- With indexes on `id`: approximately **O(n)**
- Without indexes: may approach **O(n Ă— m)** depending on the database engine and execution plan.

## Common Mistakes

### âťŚ Using INNER JOIN

```sql
SELECT unique_id, name
FROM Employees
INNER JOIN EmployeeUNI
ON Employees.id = EmployeeUNI.id;
```

This removes employees without a matching `unique_id`.

---

### âťŚ Joining on the wrong column

```sql
ON Employees.name = EmployeeUNI.id
```

Always join using the related key:

```sql
ON Employees.id = EmployeeUNI.id
```

---

### âťŚ Forgetting table aliases

When multiple tables contain the same column name (`id`), writing:

```sql
SELECT id
```

causes ambiguity.

Correct:

```sql
SELECT e.id
```

---

## Interview Tips

- Know the difference between **INNER JOIN** and **LEFT JOIN**.
- Use **LEFT JOIN** when all rows from the primary table must appear, even without a match.
- Expect interviewers to ask what changes if you replace `LEFT JOIN` with `INNER JOIN`.
- Using table aliases (`e`, `eu`) makes queries shorter and easier to read.

# Key Takeaways

- `LEFT JOIN` returns all rows from the left table.
- Missing matches are represented as `NULL`.
- The `ON` clause specifies the join condition.
- Table aliases improve readability.
- Choose `LEFT JOIN` whenever unmatched rows from the primary table must still be included.
