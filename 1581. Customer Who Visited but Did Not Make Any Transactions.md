# 1581. Customer Who Visited but Did Not Make Any Transactions

## Concept
- `LEFT JOIN`
- Filtering `NULL` values
- `GROUP BY`
- Aggregate function: `COUNT()`

## Solution

```sql
SELECT v.customer_id,
       COUNT(v.visit_id) AS count_no_trans
FROM Visits v
LEFT JOIN Transactions t
ON v.visit_id = t.visit_id
WHERE t.transaction_id IS NULL
GROUP BY v.customer_id;
```

---

## Key Points
- Use **`LEFT JOIN`** to keep all visits.
- Visits without a transaction will have `NULL` in `transaction_id`.
- Filter those visits using:
  ```sql
  WHERE t.transaction_id IS NULL
  ```
- `GROUP BY` customer to count how many visits had no transactions.

---

## SQL Clauses Used

| Clause | Purpose |
|---------|---------|
| `LEFT JOIN` | Keep all visits, even if no transaction exists |
| `ON` | Match visits with transactions |
| `WHERE ... IS NULL` | Select only visits without transactions |
| `GROUP BY` | Group records by customer |
| `COUNT()` | Count visits without transactions |

---

## Syntax

### LEFT JOIN + NULL Filter

```sql
SELECT *
FROM table1 t1
LEFT JOIN table2 t2
ON t1.id = t2.id
WHERE t2.id IS NULL;
```

### GROUP BY with COUNT

```sql
SELECT column,
       COUNT(*)
FROM table
GROUP BY column;
```

---

## Common Mistakes
- ❌ Using `INNER JOIN` (removes visits with no transactions).
- ❌ Writing `= NULL` instead of `IS NULL`.
- ❌ Grouping by `visit_id` instead of `customer_id`.
- ❌ Counting `transaction_id` (it is `NULL` after filtering).

---

## Interview Tip
Whenever the question asks:
- "Find records **without** a match"
- "Customers who **did not**..."
- "Rows **not present** in another table"

Think of:

```sql
LEFT JOIN + WHERE right_table.column IS NULL
```

This is one of the most common SQL interview patterns.

---

## Pattern Learned

```sql
SELECT ...
FROM A
LEFT JOIN B
ON A.id = B.id
WHERE B.id IS NULL;
```

> **Use this pattern to find records in one table that have no matching record in another table.**

## Key Takeaways
- `LEFT JOIN` keeps all rows from the left table.
- `IS NULL` identifies rows with no matching record.
- Use `GROUP BY` with `COUNT()` to aggregate results.
- **Revision Pattern:** `LEFT JOIN + IS NULL + GROUP BY + COUNT()`
